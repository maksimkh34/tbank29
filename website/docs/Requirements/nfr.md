# Нефункциональные требования

## Безопасность

### Защищенные соединения
- **ID**: NFR-SEC-001
- **Название**: Защищенная передача данных.
- **Описание**: Все взаимодействия между клиентом и сервером должны проводиться по защищенному соединению. Все взаимодействия между клиентом/сервером и банковскими сервисами должны проводиться в соответствии с требованиями банковского API. Перед каждой поставкой должен проводиться аудит безопасности.
- **Приоритет**: Must
- **Обоснование**: Приложение затрагивает данные о финансах. Для получения доверия пользователей и минимизации репутационных рисков нужно обеспечить приватность и безопасность данных.

### Зашифрованное хранилище
- **ID**: NFR-SEC-002
- **Название**: Защищенное хранение данных.
- **Описание**: Все чувствительные данные должны храниться зашифрованно как на стороне клиента, так и на стороне сервера.
- **Приоритет**: Must
- **Обоснование**: Обеспечение приватности и безопасности банковских транзакций и данных пользователя.

## Надежность

### Uptime
- **ID**: NFR-REL-001
- **Название**: Процент доступности сервера.
- **Описание**: Доступность сервера должна быть $\geq95\%$ (допустимый простой в месяц — 36 часов).
- **Приоритет**: Must
- **Обоснование**: Приложение делает упор на ИИ-ассистента и автодобавление банковских транзакций, которые требуют связи с сервером.

## Емкость

### Вместимость данных
- **ID**: NFR-CPC-001
- **Название**: Хранение данных
- **Описание**: Минимально допустимая вместимость на серверах — 7500 пользователей по 5000 транзакций и 100 незавершенных банковских транзакций на пользователя без потери производительности.
- **Приоритет**: Must
- **Обоснование**: Обеспечение масштабируемости системы при росте базы пользователей после MVP.

## Наблюдаемость

### Метрики
- **ID**: NFR-OBS-001
- **Название**: Сбор и хранение метрик сервера
- **Описание**: Сервер должен отслеживать собственные метрики и уведомлять администратора о внештатных ситуациях (наплыв пользователей, рост ответов 5xx).
- **Приоритет**: Should

## Совместимость

### Кроссплатформенность
- **ID**: NFR-CMP-001
- **Название**: Поддержка Android & iOS.
- **Описание**: Приложение должно работать на обеих платформах.
- **Приоритет**: Must
- **Обоснование**: Охват всей целевой аудитории пользователей смартфонов.
